# Управление задачами

Репозиторий использует файл `task-list.require.json` в качестве единственного источника данных для регистрации задач.

Каждая задача должна иметь уникальное имя в следующем формате:

```txt
001_event_loop_profiling
```

Где:

- первые три цифры — постоянный идентификатор задачи;
- остальная часть имени — человекочитаемый идентификатор (slug).

Пример:

```json
["001_event_loop_profiling", "002_async_queue", "003_worker_threads"]
```

---

## Синхронизация задач

После изменения файла `task-list.require.json` необходимо выполнить:

```bash
npm run tasks:sync
```

Во время синхронизации система выполняет следующие действия:

1. Создает директорию реализации задачи в `implements/`, если она отсутствует.
2. Создает директорию исследования задачи в `research/`, если она отсутствует.
3. Создает минимальный набор файлов для новой задачи.
4. Автоматически обновляет таблицу задач в `TRACKER.md`.
5. Сохраняет текущие статусы задач (`[ ]` и `[x]`).
6. Переименовывает директории задач при изменении имени задачи.
7. Удаляет директории задач, которые были удалены из `task-list.require.json`.

---

## Структура новой задачи

После синхронизации для новой задачи создаются следующие директории:

```txt
implements/
└── 001_event_loop_profiling
    ├── README.md
    ├── execute.sh
    ├── tsconfig.json
    └── src
        └── index.ts

research/
└── 001_event_loop_profiling
    └── index.md
```

### implements

Содержит реализацию задачи.

Основная точка входа:

```txt
src/index.ts
```

Скрипт запуска:

```txt
execute.sh
```

### research

Содержит исследование, заметки, ссылки на источники, выводы и материалы, относящиеся к задаче.

Основной файл:

```txt
index.md
```

---

## Реестр задач

Файл `TRACKER.md` поддерживается автоматически.

Пример:

```md
| ID  | Status | Task                 | Implementation                        | Research                            |
| --- | ------ | -------------------- | ------------------------------------- | ----------------------------------- |
| 001 | [ ]    | event loop profiling | ./implements/001_event_loop_profiling | ./research/001_event_loop_profiling |
```

Описание колонок:

| Поле           | Описание                        |
| -------------- | ------------------------------- |
| ID             | Уникальный идентификатор задачи |
| Status         | Статус выполнения               |
| Task           | Название задачи                 |
| Implementation | Ссылка на реализацию            |
| Research       | Ссылка на исследование          |

---

## Статусы задач

Статус задачи изменяется вручную в файле `TRACKER.md`.

Доступные значения:

```txt
[ ] - задача не выполнена
[x] - задача выполнена
```

Пример:

```md
| 001 | [ ] | event loop profiling | ... |
```

После завершения:

```md
| 001 | [x] | event loop profiling | ... |
```

Во время синхронизации статус сохраняется автоматически.

---

## Выполнение задачи

Для запуска задачи используется единая команда:

```bash
npm run execute -- 001
```

Где:

```txt
001
```

— идентификатор задачи.

Алгоритм запуска:

1. Выполняется поиск директории в `implements/`, начинающейся с указанного идентификатора.
2. Выполняется поиск файла `execute.sh`.
3. Выполняется запуск найденного скрипта.
4. Результат работы выводится в консоль.

Таким образом каждая задача может иметь собственную реализацию запуска, при этом все задачи используют общие зависимости из корневого `node_modules`.

## Жизненный цикл задачи

### Создание

1. Добавить новую запись в `task-list.require.json`.
2. Выполнить:

```bash
npm run tasks:sync
```

3. Заполнить реализацию в `implements/<task>`.
4. Выполнить исследование в `research/<task>`.

### Выполнение

Запустить задачу:

```bash
npm run execute -- <task-id>
```

Пример:

```bash
npm run execute -- 001
```

### Завершение

После завершения работы над задачей изменить статус в `TRACKER.md`:

```txt
[ ] -> [x]
```

После этого задача считается завершенной и остается в реестре для дальнейшего аудита и повторного изучения.

---

## Завершение задачи / Complete Task

Чтобы отметить задачу как выполненную, используется команда:

```bash
npm run tasks:complete -- 001
```

Где `001` — идентификатор задачи.

Команда выполняет следующие действия:

1. Находит задачу в `TRACKER.md` по ID.
2. Меняет статус задачи на `[x]`.
3. Форматирует `TRACKER.md` через Prettier.

Пример:

```md
| 001 | [ ] | event loop profiling | ./implements/001_event_loop_profiling | ./research/001_event_loop_profiling |
```

После выполнения команды:

```md
| 001 | [x] | event loop profiling | ./implements/001_event_loop_profiling | ./research/001_event_loop_profiling |
```
